01

🤔 为什么 Android 需要 CLI？

传统痛点

  • Android Studio 启动慢、吃内存
  • CI/CD 流程依赖 IDE 不现实
  • AI Agent 需要轻量交互入口
  • 跨平台脚本化操作困难

CLI 的价值

  • 单一入口，统一管理所有 Android 操作
  • 可脚本化、可组合、可自动化
  • 为 AI Agent 提供标准化工具接口
  • 轻量、快速、无 GUI 依赖
💡 核心洞察： Android CLI 不是替代 Android Studio，而是为 AI Agent 时代准备的开发者工具链基础设施。
02

🗺️ Android CLI 全景

命令 功能 场景
android create 从模板创建新项目 快速原型、脚手架
android run 部署并运行 APK 调试、测试
android sdk SDK 包管理 环境配置、版本切换
android emulator 模拟器生命周期管理 CI/CD、自动化测试
android layout UI 布局树（JSON） UI 调试、回归检测
android screen 截图 + 视觉定位 UI 文档、自动化操作
android docs 离线文档检索 AI Agent 知识库
android studio 与 Android Studio 交互 符号查找、Compose 预览
03

⚡ 一个完整工作流

# 1. 环境检查
$ android info

# 2. 安装所需 SDK
$ android sdk install platforms/android-35 build-tools/35.0.0

# 3. 创建项目
$ android create empty-activity --name="Demo" --output=./demo

# 4. 创建并启动模拟器
$ android emulator create pixel_6
$ android emulator start pixel_6_API_35

# 5. 部署运行
$ android run --debug --device=emulator-5554

# 6. 调试 UI — 获取布局树 JSON
$ android layout --pretty

# 7. 截图并标注
$ android screen capture -a -o debug.png
✓ 无需打开 Android Studio ✓ 全部命令行完成 ✓ 可脚本化、可自动化

🔍 Layout：比截图更快的 UI 调试

传统方式

截图 → 肉眼观察 → 猜测问题

慢、主观、难以精确对比

Layout 方式

JSON 布局树 → 程序化分析 → 精确 diff

快、客观、可自动化

// layout 输出示例 (简化)
{
  "class": "FrameLayout",
  "bounds": [0, 0, 1080, 2400],
  "children": [
    {
      "class": "LinearLayout",
      "bounds": [0, 84, 1080, 2316],
      "children": [...]
    }
  ]
}

核心能力

android layout

获取完整 UI 布局树，JSON 格式，可编程分析

android layout --diff

仅返回自上次 dump 以来变化的元素，精准定位 UI 变更

📚 知识库系统：AI Agent 的离线大脑

4,882 文档数量
65 MB 压缩后大小
0.38% 索引/数据比
ms 级 检索延迟

双文件模型

📄 JSON 元数据

title / short_description / keywords / url

→ 被 Lucene 索引，用于检索

📝 Markdown 正文

完整的文档内容，GitHub-Flavored Markdown

→ 按需从 zip 提取，供 LLM 消费

设计精髓： 索引只存元数据（246KB），正文留在 zip 中按需读取。搜索引擎只碰元数据，LLM 只碰正文，各取所需。

🔗 kb:// URI 协议

search 和 fetch 之间的协商层

# Step 1: 搜索 — 返回 kb:// URL
$ android docs search "Jetpack Compose text"
1. Text in Compose
   URL: kb://android/jetpack/compose/text
   Learn how to display and style text...

# Step 2: 获取 — 通过 kb:// URL 拉取正文
$ android docs fetch kb://android/jetpack/compose/text
# → 从 dac.zip 提取 .json + .md.txt → 格式化输出

解耦

search 和 fetch 各自独立演化，通过 URL 通信

可替换

后端可切换（本地 zip / 远程 HTTP / 内存），URI 不变

LLM 友好

语义清晰、可读，可直接放入 prompt 上下文

一致性保障

SHA-256 哈希驱动索引重建，确定性、无竞态

🧩 Skills 系统：AI Agent 的能力扩展

Skills 是领域知识包，可安装到多种 AI Agent 中

# 查看可用技能
$ android skills list
android-cli, adaptive, camera1-to-camerax, r8-analyzer,
edge-to-edge, jetpack-compose-m3, navigation-3, ...

# 全量安装（支持 Claude / Cursor / OpenCode / Copilot 等）
$ android skills add --all

# 按关键词搜索
$ android skills find "compose"

已支持的 Agent

Claude Cursor OpenCode Copilot Codex Qwen Continue
🔮 展望： Skills + CLI = Agent 的 Android 开发工具链。Agent 不再需要"猜"怎么操作 Android，CLI 提供标准化接口，Skills 提供领域知识。
